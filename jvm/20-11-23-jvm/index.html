jvm

JVM은 표준화된 정의로만 존재하며, Oracle, IBM 등 각 JVM 벤더들이 표준에 맞도록 별도로 JVM을 구현한 독자적인 Runtime instance라 할 수 있다. 따라서, JVM의 역할은 개발자들이 작성한 Java 프로그램이나 Web Application Server 등 구별없이 Java 프로그램의 범주에 들어가는 모든 것들을 실행시키는 데몬

Java 프로그램 실행 시

  1. complie로 class 파일 생성
  2. 해당 class 파일을 JVM으로 로딩
  3. byteCode를 해석(interpret)
  4. 메모리 등의 리소스를 할당하고 관리하며 정보 처리.
    • 이때, JVM은 thread 관리 및 Garbage Collection과 같은 메모리 정리 작업도 수행
